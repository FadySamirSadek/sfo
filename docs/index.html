What is sfo?

sfo is an instant build process. Rather than having to manage your configuration of tools like Webpack, Babel, ESLint, and others, sfo governs these tools for you.

How to use sfo

sfo will take care of all the build process for both JavaScript and HTML, generating your index.html, and your linting as well. All you need to do is write an entry point file that exports a default function that does your rendering. From there sfo will do the rest.

sfo is not tied to any one framework. sfo is built to handle projects in frameworks like React, Angular, and Vue.

Usage

Get Started

npm install --global sfo
sfo dev [your script file]

Commands

sfo dev [your script file]

Run sfo in dev mode. Get all the useful features of Webpack's dev server.

sfo bundle [your entry file]

Bundles for production. Minifies, tree shakes, copies in files from your public directory, ready for production. Outputs to ./build/.

sfo size [your entry file]

Visualize your Webpack bundle so you can see where your file size is coming from.

Flags

--help, -h

See help message.

--version, -v

See version of sfo.

--no-eslint, -n

Does not run ESLint against your code.

--title "[your title]", -T "[your title]"

Changes the title of in the head of your index.html.

Shadowing

Does one part of sfo not satisfy your requirements? sfo allows you to replace only the parts you need to. Both your .babelrc and index.html can be overwritten; just write your file with either of those names and sfo will use those instead of its own files.
